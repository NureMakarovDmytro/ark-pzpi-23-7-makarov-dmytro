2 ЗАВДАННЯ
 Метою даної практичної роботи є дослідження та аналіз офіційних
рекомендацій щодо оформлення програмного коду для обраної мови
програмування, а також демонстрація їх практичного застосування на
конкретних прикладах.
У рамках роботи необхідно обрати мову програмування, ознайомитися з
її офіційними coding conventions та підготувати презентацію з детальними
поясненнями кожної рекомендації.
 Студент повинен здійснити дослідження офіційної документації
обраної мови програмування, зокрема style guides та coding conventions.
Необхідно виділити ключові аспекти оформлення коду: правила іменування
(пакетів, класів, функцій, змінних, констант), принципи форматування
(відступи, дужки, пробіли), структуру класів та файлів, ідіоматичні
конструкції мови, а також рекомендації щодо написання читабельного та
підтримуваного коду. Для кожної рекомендації необхідно підготувати наочні
приклади коду, які демонструють як правильне, так і неправильне
застосування відповідного правила, з детальними поясненнями важливості
кожного стандарту.
 На основі проведеного дослідження студент має підготувати
презентацію, яка включає вступну частину з оглядом обраної мови
програмування, основну частину з детальним розглядом конкретних
рекомендацій coding conventions із прикладами коду, а також висновки про
важливість дотримання стандартів оформлення. Презентація повинна містити
не менше 10 основних слайдів, бути оформленою українською мовою та
містити чіткі приклади коду.

3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Вибір мови програмування та джерел
Для виконання практичної роботи обрано мову програмування Pascal –
мову для структурованого та об'єктно-орієнтованого програмування,
розроблену Ніклаусом Віртом у 1970 році. Сучасні варіанти: Object Pascal
(Delphi) та Free Pascal.
3.2 Правила іменування
Згідно з Object Pascal Style Guide, units іменуються в нижньому регістрі
без підкреслень (myunit.pas), класи/типи – у стилі PascalCase з префіксом T
(TUserManager). Функції та процедури використовують PascalCase
(ProcessUser), змінні – camelCase (userName), константи – UPPER_CASE
(MAX_VALUE).
 1 unit My_Unit
 2 type
 3 tuser_manager = class;
 4 t_data_processor = class;
 5 end.
 6 unit myunit
 7 type
 8 TUserManager = class;
 9 TDataProcessor = class;
 10 end.
 3.3 Правила форматування коду
 Офіційні conventions визначають використання 2-4 пробілів для
відступів, begin/end на нових рядках, пробіли навколо операторів (:=, +, =),
але не навколо . чи [].
5
 1 procedure Calculate;
 2 begin if Condition then begin
 3 DoSomething();
 4 DoMore(); end end;
 5 end;
 6 var Sum:=A+B; Result=A+B*C; Name=Person.Name;
 7 procedure Calculate;
 8 begin
 9 if Condition then
 10 begin
 11 DoSomething();
 12 DoMore();
 13 end;
 14 end;
 15 var
 16 Sum := A + B;
 17 Result := A + B * C;
 18 Name := Person.Name;
 3.4 Структура класу
 Object Pascal Style Guide визначає порядок елементів у unit: interface
(uses, const, type, var, procedure declarations), implementation (uses, var,
procedure bodies), initialization. Цей порядок забезпечує логічну послідовність
від оголошення до реалізації.
 1 unit MyUnit;
 2 interface
 3 uses
 4 SysUtils;
6
 5 const
 6 MAX_VALUE = 100;
 7 type
 8 TUserManager = class;
 9 var
 10 GlobalVar: Integer;
 11 procedure ProcessUser();
 12 implementation
 13 uses
 14 Classes;
 15 var
 16 ImplVar: string;
 17 procedure ProcessUser();
 18 begin
 19 // Логіка
 20 end;
 21 initialization
 22 // Ініціалізація
 23 end.
 3.5 Ідіоматичний Pascal
 Conventions рекомендують використовувати const замість var для
незмінних даних, уникати глобальних змінних. Для функцій з одним виразом
використовуйте пряме присвоєння :=.
 1 var
 2 Name: string = 'John';
 3 Config: TConfig;
 4 function Double(X: Integer): Integer;
 5 begin
7
 6 Result := X * 2;
 7 end;
 8 const
 9 Name = 'John';
 10 DEFAULT_TIMEOUT = 30;
 11 var
 12 Counter: Integer;
 13 function Double(X: Integer): Integer;
 14 begin
 15 Double := X * 2;
 16 end;
 3.6 Параметри за замовчуванням
 Object Pascal Conventions рекомендують перевантаження (overloads)
замість множини сигнатур для симуляції параметрів за замовчуванням.
 1 procedure Connect(Host: string);
 2 begin
 3 Connect(Host, 8080);
 4 end;
 5 procedure Connect(Host: string; Port: Integer);
 6 begin
 7 // Логіка
 8 end;
 9 procedure Connect(Host: string; Port: Integer = 8080);
 10 overload;
 11 procedure Connect(Host: string);
 12 begin
 13 Connect(Host, 8080);
 14 end;
8
 15 Connect('localhost');
 3.7 Умовні оператори
 Conventions визначають використання if для бінарних умов та case
для трьох і більше варіантів.
 1 if IsValid then
 2 ProcessData()
 3 else
 4 ShowError();
 5 var Result: string;
 6 if Success then Result := 'OK' else Result := 'Error';
 7 if IsValid then
 8 ProcessData()
 9 else
 10 ShowError();
 11 case Type of
 12 JSON: Result := ParseJson();
 13 XML: Result := ParseXml();
 14 else Result := ParseDefault();
 15 end;
 3.8 Інструменти автоматизації
 Для дотримання Object Pascal Style Guide використовуються: Pascal
Analyzer (статичний аналіз), IDE formatters в Delphi/Free Pascal, GExperts
(плагіни для рефакторингу).

ВИСНОВКИ
 У результаті виконання практичної роботи було розглянуто офіційні
Object Pascal Style Guide, що визначають стандарти оформлення програмного
коду для мови Pascal. Досліджено ключові аспекти conventions: правила
іменування units, типів, функцій, змінних та констант; принципи
форматування коду з відступами, дужками та пробілами; структуру units;
ідіоматичні підходи Pascal (принцип незмінності, пряме присвоєння,
перевантаження); рекомендації щодо умовних операторів.
 Практична робота показала, що використання офіційних стандартів
оформлення коду та інструментів автоматизації (Pascal Analyzer, IDE
formatters) є необхідною умовою для професійної розробки програмного
забезпечення.
